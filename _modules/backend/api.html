async api.main.classify_plant(image_data)[source]

Endpoint for classifying multiple Base64-encoded images of plants.

Parameters:

image_data (dict) – JSON data containing a list of Base64-encoded image strings.

Returns:

A dictionary containing results for each processed image, including recognized plant names,

additional data, and probabilities.

Return type:

dict

Raises:

  • HTTPException (400) – If the ‘images’ field is missing in the request.

  • HTTPException (500) – For unexpected errors in processing or classification.

Example request:
{
“images”: [

“data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAAAAAAAD/2wBDAP8A/wD/…”, “data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAAAAAAAD/2wBDAP8A/wD/…”]

}

Example response:
{
“results”: [
{

“image”: “data:image/jpeg;base64,/9j/4AAQSkZJRgABAQEAAAAAAAD/2wBDAP8A/wD/…”, “recognitions”: [

{

“name”: “Rosa indica”, “plant”: { … }, # Additional plant data “wikipedia”: “https://en.wikipedia.org/wiki/Rosa_indica”, “probability”: 0.98

}, {

“name”: “Rosa rugosa”, “plant”: { … }, # Additional plant data “wikipedia”: “https://en.wikipedia.org/wiki/Rosa_rugosa”, “probability”: 0.85

}

]

}

]

}

api.main.decode_and_save_image(image_base64)[source]

Decodes a Base64-encoded image and saves it as a JPG file in the upload folder.

Parameters:

image_base64 (str) – The Base64-encoded image string (can include data URL prefix).

Returns:

The absolute file path of the saved JPG image.

Return type:

str

Raises:

  • HTTPException (413) – If the image size exceeds the configured limit.

  • HTTPException (400) – If the Base64 format is invalid or the image format is unsupported.

  • HTTPException (500) – If an unknown error occurs during processing.

async api.main.search_plant(plant_data)[source]

Endpoint for searching plants by their names and retrieving additional data.

Parameters:

plant_data (dict) – A dictionary containing the plant name(s) under the “name” key. The “name” key can contain a single plant name (string) or a list of plant names.

Returns:

A dictionary containing the results of the search with additional plant information.

The results include the plant name, its associated data, and a link to the Wikipedia page.

Return type:

dict

Raises:

HTTPException (400) – If the ‘name’ field is missing in the request or if it’s an invalid input.

Example request:
{

“name”: “Rosa indica”

}

Example response:
{
“results”: [
{

“name”: “Rosa indica”, “plant”: { … }, # Additional plant data “wikipedia”: “https://en.wikipedia.org/wiki/Rosa_indica”

}

]

}
